# Prompt for LLM: Update tags in models.yaml

## Task Overview

Your task is to update the `tags` field for all models in `models.yaml`. 
The `models.yaml` file has been automatically generated by a script and contains the latest models from OpenRouter, but it is missing descriptive tags.

## Step 1: Run the Update Script

**IMPORTANT**: You must first run the fetch script to generate the base `models.yaml` file:

```bash
pnpm update-models > models.yaml
```

This script handles:
- Filtering models (free, vision, tools, json support)
- Grouping by provider and sorting

## Step 2: Add Tags to Models

Now, read the generated `models.yaml` and for each model, add or update the `tags` array based on the following rules.

### Tagging Rules

Generate tags based on model characteristics:

**1. Base Tags**:
- `general`: Include for general-purpose models (most models)

**2. Capability Tags**:
- `code`: If model ID contains 'code', 'coder', 'codestral', 'devstral'
- `reasoning`: If model ID contains 'reasoning', 'r1', 'think', 'deepresearch'
- `vision`: If model has `supportsVision: true` (which should be true for all models in the script output)
- `paid`: If the model is not free (the script currently filters for free models, but if any paid models appear, add this tag)

**3. Family/Version Tags**:
- Extract the actual provider/creator and major version from model ID.
- Create a tag in the format `provider-version`.
- Examples:
  - `meta-llama/llama-3.3-70b-instruct` → add tag `meta-llama-3`
  - `google/gemini-2.0-flash-exp` → add tag `google-gemini-2`
  - `mistralai/mistral-7b-instruct` → add tag `mistral-7b`
  - `qwen/qwen-2.5-72b-instruct` → add tag `qwen-2`
  - `anthropic/claude-3-5-sonnet` → add tag `anthropic-claude-3`

**4. Additional Logic**:
- Assign `weight` (priority) if it seems wrong (1-10 range). Premium models like Llama 3.3 70B or Claude 3.5 should have higher weights (e.g., 10).
- Ensure `available: true` and `jsonResponse: true`.

## Requirements

- Provide the COMPLETE updated `models.yaml` file.
- Keep the comments and grouping created by the script.
- Ensure valid YAML syntax.
- Do not remove any models that were generated by the script.
  - **reasoning**: If model ID contains 'reasoning', 'r1', 'think'
  - **fast**: < 20B parameters or "flash"/"mini"/"lite"
  - **medium**: 20B-100B parameters
  - **slow**: > 100B parameters or reasoning models

## Example Entry

```yaml
- name: llama-3.3-70b-instruct
  provider: openrouter
  model: meta-llama/llama-3.3-70b-instruct:free
  type: fast
  contextSize: 131072
  maxOutputTokens: 4096
  tags:
    - general
    - meta-llama-3
  jsonResponse: true
  available: true
  weight: 10
  supportsVision: true
```

## Output Format

Output the full content of the updated `models.yaml` file.
